## 📊 數據輸入規範（更新）

### ✅ **正確嘅數據格式**
- **時間格式**：`HH:MM:SS` 或 `H:MM:SS`（06:15:00 或 3:30:00）
- **司機資訊**：完整姓名+車牌+電話，用空格分隔
- **訂單重複**：同一個訂單號可能有多行（雙車輛服務）
- **服務類型**：點對點接送、On Hire 6hrs、雙車輛等

### 📋 **欄位說明**
- **Ord No.**：訂單編號（數字）
- **Time**：時間格式（HH:MM:SS 或 H:MM:SS）
- **Trip**：路線描述（如 TST > AP）
- **Driver**：司機資訊（姓名+車牌+電話）
- **Status**：狀態（可留空）
- **Pick**：上車時間
- **Drop**：下車時間
- **Decp**：附加費用（數字）
- **Ptime**：預計時間（可留空）
- **Driver.Details**：備註信息（如 2 cars, 急單）
- **Dvr.OT**：加班費（數字）
- **Dvr.C**：基本車費（通常400）
- **Driver.C+OT**：總車費（基本+加班）
- **Bouns**：獎金（通常30）

### ✅ **數據輸入標準**
1. **時間格式**：支援 `06:15:00` 或 `3:30:00` 兩種格式
2. **司機資訊**：完整格式 `姓名 車牌 電話`（如 `Eva TZ9800 97062468`）
3. **訂單重複**：同一訂單號可能有多行（如雙車輛服務）
4. **車費計算**：`Dvr.C + Dvr.OT = Driver.C+OT`
5. **獎金標準**：通常係30
6. **附加費**：`Decp`用數字表示

## 🚗 服務類型說明

### 📋 **常見服務類型**
1. **點對點接送**：`TST > AP`, `AP > Cordis`
2. **雙車輛服務**：同一訂單兩行記錄，兩個司機
3. **On Hire 6hrs**：包車服務，如 `K11 Onhire 6hrs`
4. **急單**：喺 `Driver.Details` 標註 `急單`
5. **附加服務**：如 `+30 mins buffer time`

### 💰 車費計算規則
- **基本車費**：`Dvr.C = 400`
- **加班費**：`Dvr.OT` 根據情況設定
- **總車費**：`Driver.C+OT = Dvr.C + Dvr.OT`
- **附加費**：`Decp` 另外計算（如機場附加費）
- **獎金**：`Bouns = 30`
