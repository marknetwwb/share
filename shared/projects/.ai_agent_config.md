# AI Agent 配置文件

## 🤖 Agent 配置說明

### 系統要求
- 所有AI必須運行喺 OpenClaw 環境
- Agent 唔可以直接訪問 `/data/workspace/shared/` 以外嘅目錄
- 必須遵守訪問政策

### Agent 類型定義

#### 1. 主管理AI
- **名稱**: Main Coordinator
- **職責**: 任務分配、資源管理、協調統籌
- **訪問權限**: 完全訪問所有共享目錄
- **工作目錄**: `/data/workspace/shared/`

#### 2. 小說創作AI
- **名稱**: Novel Creator
- **職責**: 小說內容創作、角色設計、情節構建
- **訪問權限**: 訪問 `projects/novel/` 同相關文檔
- **工作目錄**: `/data/workspace/shared/projects/novel/`

#### 3. 論壇管理AI
- **名稱**: Forum Manager
- **職責**: HKGBook論壇活動、回覆管理、內容創作
- **訪問權限**: 訪問 `projects/forum/` 同相關文檔
- **工作目錄**: `/data/workspace/shared/projects/forum/`

#### 4. 系統維護AI
- **名稱**: System Maintainer
- **職責**: 系統監控、錯誤處理、性能優化
- **訪問權限**: 訪問 `projects/system/` 同系統日誌
- **工作目錄**: `/data/workspace/shared/projects/system/`

## 🔧 Agent 啟動配置

### 環境變數
```bash
SHARED_WORKSPACE=/data/workspace/shared
PROJECT_ROOT=/data/workspace/shared/projects
LOG_DIR=/data/workspace/shared/logs
```

### 起動參數
- 所有AI必須指定 `--workspace=/data/workspace/shared`
- 必須設置 `--agent-type` 參數
- 必須配置 `--access-level` 參數

## 📋 Agent 狀態監控

### 狀態檢查點
1. 啟動時驗證訪問權限
2. 工作時檢查資源使用
3. 完成後更新工作日誌
4. 錯誤時自動報警

### 健康檢查
- CPU 使用率監控
- 記憶體使用監控
- 磁碟空間監控
- 網絡連接監控

## 🚨 緊急處理

### Agent 失敗處理
1. 自動重啟機制
2. 任務重新分配
3. 錯誤日誌記錄
4. 人工介入通知

### 資源不足處理
1. 任務優先級調整
2. 臨時資源回收
3. 用戶通知機制
