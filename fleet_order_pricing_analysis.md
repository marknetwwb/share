### 📋 **價格決定因素**

#### 🚗 **服務類型**
1. **點對點接送**：基本400 + 60加班費 = 460
2. **長途路線**：基本400 + 95加班費 = 495
3. **附加服務**：基本400 + 附加費 = 調整後價格
4. **急單**：基本460 + 加班費 = 595
5. **On Hire包車**：根據時長收費 = 1380起

#### ⏰ **時間因素**
- **日間常規**：基本400 + 60 = 460
- **長途路線**：基本400 + 95 = 495
- **夜間/高峰期**：可能附加加班費
- **急單**：額外附加費460起

#### 📍 **路線因素**
- **市區內**：TST > AP = 460
- **跨區路線**：MB179 > TST = 495
- **機場/特殊地區**：可能收附加費
- **包車服務**：按時長計費

### 🔍 **驗證規則**

#### ✅ **計算公式**
```
Driver.C+OT = Dvr.C + Dvr.OT
```
所有計算都正確，唔會有數學錯誤。

#### ✅ **獎金標準**
- 大多數訂單：Bouns = 30
- 空白記錄：Bouns = 0

#### ❌ **無附加費記錄**
- Decp 欄位大多係空白
- 可能其他方式處理附加費

## 🎯 **學習重點**

### ✅ **正確做法**
1. **唔好假設固定價格**：Dvr.C 可能係 400, 460, 1380 等
2. **理解服務類型**：根據服務決定價格
3. **驗證計算**：確保 Driver.C+OT = Dvr.C + Dvr.OT
4. **注意特殊服務**：急單、包車價格唔同

### ❌ **避免嘅錯誤**
1. **以為所有Dvr.C都係400**
2. **忽略服務類型對價格嘅影響**
3. **唔驗證車費計算**
4. **以為附加費喺Decp欄位**

## 💡 **總結**

價格設定係動態嘅，根據：
- **服務類型**（點對點、長途、急單、包車）
- **路線長短**（市區內、跨區、機場）
- **時間因素**（日間、夜間、高峰期）
- **特殊需求**（急單、附加服務）

日後處理訂單時，一定要睇清楚每個訂單嘅具體價格設定，唔好以為係固定數字。
